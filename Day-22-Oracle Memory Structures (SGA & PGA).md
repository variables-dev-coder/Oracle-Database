# Oracle Memory Architecture

Oracle allocates memory when the database instance starts.

```
                Oracle Memory
                     |
        ----------------------------
        |                          |
       SGA                        PGA
 (Shared Memory)           (Private Memory)
```

---

# What is SGA?

**SGA (System Global Area)** is a shared memory region allocated when the Oracle Instance starts.

It stores information that is shared among all users connected to the database.

### Key Points

- Shared by all users
- Allocated during instance startup
- Improves database performance
- Reduces disk I/O
- Stores frequently accessed data and SQL information

---

# Why is SGA Important?

Without SGA:

- Every SQL query would access the disk.
- Disk access is slow.
- Database performance decreases.

With SGA:

- Frequently accessed data remains in memory.
- SQL execution becomes faster.
- Disk access is reduced.

# Components of SGA

```
                 SGA
                  |
------------------------------------------------
|             |             |        |          |
Buffer Cache  Shared Pool  Redo Log  Large Pool Java Pool
```

---

# 1. Database Buffer Cache

## Purpose

Stores recently accessed database blocks in memory.

Oracle first checks the Buffer Cache before reading from disk.

### Example

```sql
SELECT * FROM employees;
```

First execution:

```
Disk
  |
  V
Buffer Cache
```

Second execution:

```
Buffer Cache
      |
      V
Result
```

No disk access is required.

### Advantages

- Faster query execution
- Less disk I/O
- Better overall performance

### Interview Answer

**What is Buffer Cache?**

Database Buffer Cache stores recently accessed data blocks in memory to reduce physical disk reads and improve performance.

---

# 2. Shared Pool

## Purpose

Stores SQL statements, execution plans, and metadata.

It allows Oracle to reuse parsed SQL instead of parsing it repeatedly.

### Components

### Library Cache

Stores:

- SQL Statements
- PL/SQL Code
- Execution Plans

Example:

```sql
SELECT * FROM employees;
```

If another user executes the same SQL, Oracle reuses the existing execution plan.

---

### Data Dictionary Cache

Stores metadata such as:

- Table names
- Column names
- User information
- Privileges
- Object definitions

Oracle does not need to read these repeatedly from disk.

### Advantages

- Reduces SQL parsing
- Saves CPU
- Improves execution speed

### Interview Answer

**What is Shared Pool?**

Shared Pool stores parsed SQL, execution plans, PL/SQL code, and data dictionary information to improve SQL execution efficiency.

---

# 3. Redo Log Buffer

## Purpose

Stores redo entries generated by database changes.

Every database modification generates redo information.

Example:

```sql
UPDATE employees
SET salary = 50000
WHERE id = 10;
```

Flow:

```
SQL Statement
      |
Redo Log Buffer
      |
Redo Log Files
      |
Crash Recovery
```

### Advantages

- Supports database recovery
- Protects committed transactions
- Minimizes data loss

### Interview Answer

**What is Redo Log Buffer?**

Redo Log Buffer temporarily stores redo entries before Oracle writes them to the online redo log files.

---

# 4. Large Pool

## Purpose

Optional memory area used for:

- RMAN Backup
- Shared Server
- Parallel Query Execution

### Benefit

Prevents these operations from consuming Shared Pool memory.

---

# 5. Java Pool

## Purpose

Used when Oracle Java Virtual Machine (JVM) features are enabled.

Stores memory required for Java execution.

---

# SGA Summary

| Component | Purpose |
|-----------|----------|
| Buffer Cache | Stores recently accessed database blocks |
| Shared Pool | Stores SQL, execution plans, metadata |
| Redo Log Buffer | Stores redo information before writing to redo logs |
| Large Pool | Used by RMAN, Shared Server, Parallel Execution |
| Java Pool | Used for Oracle JVM |

---

# What is PGA?

**PGA (Program Global Area)** is private memory allocated to each Oracle server process.

Unlike SGA, PGA is **not shared**.

Each connected session has its own PGA.

---

# PGA Architecture

```
User A
   |
Server Process
   |
PGA

----------------------

User B
   |
Server Process
   |
PGA
```

Each process has its own private memory.

---

# Components of PGA

- Sort Area
- Session Memory
- Cursor Information
- Stack Space

---

# 1. Sort Area

Used for:

- ORDER BY
- GROUP BY
- DISTINCT
- CREATE INDEX

Example:

```sql
SELECT *
FROM employees
ORDER BY salary;
```

Sorting occurs inside the PGA.

If the data is too large, Oracle uses the Temporary Tablespace.

---

# 2. Session Memory

Stores:

- Session information
- Login details
- Session variables

Each user has separate session memory.

---

# 3. Cursor Information

Stores information about SQL cursors.

Each server process maintains its own cursor information.

---

# 4. Stack Space

Stores:

- Function calls
- Local variables
- Execution information

Similar to the Java Stack Memory.

---

# PGA Summary

| Component | Purpose |
|-----------|----------|
| Sort Area | Sorting operations |
| Session Memory | Session information |
| Cursor Information | SQL cursor management |
| Stack Space | Function calls and local variables |

---

# SGA vs PGA

| Feature | SGA | PGA |
|----------|-----|-----|
| Full Form | System Global Area | Program Global Area |
| Memory Type | Shared | Private |
| Shared Between Users | Yes | No |
| Created When | Instance Starts | Server Process Starts |
| Used By | All Sessions | Single Session |
| Stores | Cache, SQL, Redo | Sort, Cursor, Session |
| Performance | Shared Performance | Process Performance |

---

# Memory Flow Example

Query:

```sql
SELECT *
FROM employees
ORDER BY salary;
```

Execution Flow

```
User
   |
Server Process
   |
PGA
(Sorting)
   |
SGA
(Buffer Cache)
   |
Disk
```

Oracle first checks the Buffer Cache.

If the data is unavailable:

- Reads from disk
- Stores data in Buffer Cache
- Performs sorting in PGA
- Returns the result

# Interview Questions

## 1. What is SGA?

**Answer:**

SGA (System Global Area) is a shared memory region allocated when the Oracle instance starts. It stores shared data and control information used by all connected users.

---

## 2. What is PGA?

**Answer:**

PGA (Program Global Area) is private memory allocated to each Oracle server process. It stores process-specific information such as sorting, session data, cursors, and stack space.

---

## 3. What is Buffer Cache?

**Answer:**

Buffer Cache stores recently accessed database blocks in memory to reduce disk I/O and improve performance.

---

## 4. What is Shared Pool?

**Answer:**

Shared Pool stores parsed SQL statements, execution plans, PL/SQL code, and data dictionary cache.

---

## 5. What is Redo Log Buffer?

**Answer:**

Redo Log Buffer temporarily stores redo entries generated by database changes before they are written to online redo log files.

---

## 6. What is the purpose of Large Pool?

**Answer:**

Large Pool is used for RMAN backups, Shared Server, and Parallel Query Execution.

---

## 7. Is PGA shared?

**Answer:**

No.

Each Oracle server process has its own private PGA.

---

## 8. Which memory stores SQL execution plans?

**Answer:**

Shared Pool (Library Cache).

---

## 9. Which memory stores sorting operations?

**Answer:**

PGA (Sort Area).

---

## 10. What is the biggest difference between SGA and PGA?

**Answer:**

SGA is shared memory for all users, while PGA is private memory allocated to each server process.

# Key Takeaways

- **SGA** is shared memory used by all sessions.
- **PGA** is private memory for each server process.
- **Database Buffer Cache** stores frequently accessed data blocks.
- **Shared Pool** stores SQL statements, execution plans, and metadata.
- **Redo Log Buffer** stores redo entries for database recovery.
- **Large Pool** supports RMAN backups and parallel execution.
- **Java Pool** is used for Oracle JVM operations.
- **Sorting, cursors, and session data** are managed in the PGA.
- Oracle checks **Buffer Cache first**, then accesses the disk if necessary.
- Understanding SGA and PGA is fundamental for Oracle performance tuning and DBA interviews.
